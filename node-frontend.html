El rol de Node.js en el Frontend

En el desarrollo Frontend moderno (con Astro, React, Vue, Angular, Svelte, etc.), la función de Node.js se limita al entorno de desarrollo (ordenador del desarrollador) y a los servidores de despliegue (CI/CD).

Básicamente, Node.js actúa como un motor de tus herramientas de trabajo. Sirve principalmente para tres cosas:

  1. Ejecutar Gestores de Paquetes (usaremos pnpm): Permite instalar, actualizar y administrar librerías de terceros (como Tailwind, Axios, etc).
  2. Brindar un Servidor de Desarrollo Local: Al usar el comando pnpm dev desde la terminal, Node.js levanta un servidor web local para que puedas ver los cambios en tiempo real mientras programas gracias a herramientas como Vite y su sistema de reemplazo de módulos en caliente (Hot Module Replacement).
  3. Ejecutar herramientas de construcción (Bundlers/Compiladores): Herramientas como Vite o esbuild necesitan Node.js para ejecutarse. Son las encargadas de tomar un código moderno (TypeScript, JSX, CSS avanzado) y transformarlo en archivos HTML, JS y CSS simples que cualquier navegador web estándar pueda entender.

pnpm como gestor de paquetes

Si Node.js es el entorno de ejecución, pnpm es el administrador.

  • Node.js es como el sistema de producción de una fábrica.
  • pnpm es como el gerente que aprovisiona y gestiona los insumos de la fábrica.

¿Por qué pnpm?

En la actualidad (2026) pnpm tiene importantes ventajas sobre otros gestores sobre otros gestores tradicionales como npm o yarn:

  • Velocidad y Espacio (El almacén global): A diferencia de npm, que duplica los archivos en la carpeta node_modules de cada proyecto, pnpm guarda todos los paquetes en un único lugar de tu disco duro (un almacén global). Así, si 10 proyectos usan React, pnpm solo lo descarga una vez y crea enlaces (hard links) en cada proyecto. Esto ahorra gigabytes de espacio en el disco duro del programador.
  • Estructura estricta: pnpm evita que uses librerías que no has declarado explícitamente en el package.json. No declararlas causa un problema común en npm conocido como phantom dependencies (dependencias fantasmas).

Comandos esenciales de pnpm

  • pnpm install: instala todas las dependencias declaradas de un proyecto.
  • pnpm add [paquete] :Agrega una nueva librería al proyecto para producción.
  • pnpm add -D [paquete] : Agrega una librería exclusiva para desarrollo (compiladores, linters, etc.).
  • pnpm dev: arranca un servidor de desarrollo local.
  • pnpm dlx: ejecuta binarios temporales sin instalarlos (como npx).

El stack de FrontEnd

Un stack de frontend es el conjunto de lenguajes, frameworks y herramientas que utilizamos para construir una interfaz web. En este curso iremos escalando de forma guiada para conformar nuestro stack:

1. HTML + CSS (TailwindCSS v4) + JS Vanilla

Tailwind CSS es un framework de CSS de "clases de utilidad" (utility-first). En lugar de ofrecer componentes prefabricados (como botones ya diseñados), proporciona clases de bajo nivel que se aplican directamente en el HTML. Es el estándar de diseño actual.

En el pasado hemos ido utilizando Tailwind css enlazado desde un cdn. Ahora lo instalaremos directamente en el proyecto para aprovechar todo su potencial.

Para instalarlo directamente en un proyecto nativo, ejecutamos en la terminal:

pnpm add -D tailwindcss postcss autoprefixer

En Tailwind v4 ya no existe el archivo tailwind.config.js. Toda la configuración se realiza directamente mediante CSS nativo.

Desplegar el proyecto en GitHub (método manual tradicional)

Paso 1: Compilar tu código en local (Generar el CSS final)

Antes de subir nada, necesitamos que Node.js y Tailwind generen el archivo CSS real que el navegador puede leer.

  • Abre la terminal en tu editor (Visual Studio Code) y asegúrate de que tu servidor de desarrollo esté apagado (puedes presionar Ctrl + C para apagarlo)

  • Ejecuta el comando de producción:

    pnpm build

  • Verás que en tu proyecto se crea (o se actualiza) una carpeta llamada dist/ que contiene un archivo llamado output.css. ¡Ese es el CSS optimizado y minificado de Tailwind v4!

Paso 2: Crear el repositorio en GitHub

Crea un nuevo repositorio y configuralo de la siguiente manera. Una vez listo, haz clic en el botón Create Repository:

  • Repository name: dale un nombre corto al repositorio en minúsculas preferiblemente
  • Public / Private: Selecciónalo como Public (Obligatorio para que GitHub Pages sea gratis).
  • Opciones iniciales: Deja todo lo demás desmarcado (sin README, sin .gitignore).

Paso 3: Subir los archivos manualmente

Busca un enlace que dice: "uploading an existing file" (subir un archivo existente). Se abrirá una zona donde puedes arrastrar tus archivos.

⚠️ Vamos a arrastrar solo lo que el navegador necesita y lo que configura nuestro proyecto.

NUNCA arrastres la carpeta node_modules (pesa demasiado y romperá la subida).

Arrastra hacia la ventana de GitHub los siguientes archivos y carpetas:

  • Tu archivo index.html (debe quedar en la raíz).
  • Tu archivo package.json.
  • Tu carpeta dist (que creamos en el Paso 1, y que contiene el CSS final).
  • Tus archivos de JavaScript si tuvieras (script.js, etc.).

Paso 4: Encender GitHub Pages

Recuerda hacerlo como siempre: Settings > Pages > Deploy from a branch y seleccionar main (o master) donde dice none.

Deja la carpeta en / (root) y haz clic en Save (Guardar).

2. HTML - CSS+TailwindCSS - JS Vanilla+Astro.build

Astro.build: Es considerado el framework 'todo en uno'. Se basa en generar HTML estático y usar Node.js en el servidor solo cuando es estrictamente necesario.

🎻 Filosofía Astro: Astro actúa como el director de orquesta, mientras que el resto de frameworks o librerías del ecosistema son los músicos.

Para iniciar un proyecto con este stack, ejecutamos en la terminal:

pnpm create astro@latest

El asistente te guiará con unas preguntas (nombre de la carpeta, inicializar Git, etc.). Cuando te pregunte si deseas instalar las dependencias con pnpm, selecciona Sí.

Para integrar Tailwind de forma automática, aprovechamos que Astro es nuestro director de orquesta y ejecutamos:

pnpm astro add tailwind

Astro descargará tailwindcss, configurará las herramientas y activará la integración de forma limpia en tu archivo astro.config.mjs.

Durante este proceso, Astro creará un archivo CSS global (normalmente en src/styles/global.css). Al abrirlo, verás la importación de tailwind:

@import "tailwindcss";

Agregar plugins de Tailwind desde Astro

Agregaremos dos plugins en esta oportunidad:

  • @tailwindcss/forms: Tailwind por defecto (Preflight)limpia y aplana los formularios para que no rompan tu diseño, pero toca estilizarlos casi desde cero. Este plugin, modifica automáticamente el estilo de los controles de formulario, permitiéndo cambiar sus colores o bordes usando clases simples de Tailwind de forma mucho más directa:

    - Todos los inputs, selectores y textareas tienen un borde sutil, un sombreado ligero y esquinas redondeadas muy elegantes por defecto.

    - Los checkbox y radio buttons se vuelven estilizable de forma nativa.

  • tailwindcss-animate: añade decenas de clases listas para usar: desvanecimientos, deslizamientos, zooms y efectos de entrada/salida combinables.

Paso 1: Desde la terminal del proyecto Astro, añadimos los plugins como dependencias de desarrollo (-D) usando pnpm:

Desplegar el proyecto en Netlify

Paso 1. Compilar el proyecto

El proceso es similar al Stack 1.

  • Abre la terminal en tu editor (Visual Studio Code) y asegúrate de que tu servidor de desarrollo esté apagado (puedes presionar Ctrl + C para apagarlo)

  • Ejecuta el comando de producción:

    pnpm build

  • Verás que en tu proyecto se crea (o se actualiza) la carpeta llamada dist/ (el código de producción)

Paso 2. Subir el proyecto a Netlify

Netlify tiene una herramienta específica que se llama Netlify Drop.

  • Entra a app.netlify.com/drop.

    No hace falta crear repositorio. Puedes arrastrar la carpeta dist incluso sin haber iniciado sesión (aunque si no te creas una cuenta, la web caduca en 24 horas. Si te registras con tu email, el proyecto se guarda para siempre).

  • Arrastra la carpeta dist, la plataforma lee el index.html que Astro generó dentro de ella, y te dará da la URL en 3 segundos.

¿Cómo gestionar múltiples proyectos en Netlify?

La primera vez que arrastres la carpeta dist, la plataforma te creará un "Proyecto 1" con una URL aleatoria (que una vez registrado, puedes cambiar por algo como mi-proyecto-astro.netlify.app).

Cuando quieras subir tu segundo proyecto:

  • Entra a tu panel de control (Dashboard) de Netlify. Verás la lista con tu primer proyecto.
  • Haz clic en un botón que dice Add new site (Añadir nuevo sitio) ➔ Deploy manually (Desplegar manualmente).
  • Te volverá a salir un recuadro similar al que usaste para el primer proyecto. Arrastra la nueva carpeta dist de tu segundo proyecto, y tendrás un Proyecto 2 con una URL totalmente diferente.

Instalación de Node.js

Paso 1. Desinstalar el Node.js existente

Desinstalar el Node.js viejo

  • Ve al Panel de Control de Windows > Desinstalar un programa (o ve a Configuración > Aplicaciones > Aplicaciones instaladas).
  • Busca Node.js en la lista.
  • Haz clic derecho y dale a Desinstalar.

Borrar carpetas fantasma

  • Presiona las teclas Windows + R, escribe %appdata% y dale Enter.
  • Si ves una carpeta llamada npm o npm-cache, bórralas sin miedo.

Ahora, estableceremos el entorno de desarrollo. Usaremos cmdr como consola, fnm como gestor de versiones de Node.js y pnpm como gestor de paquetes.

Paso 2. Instalar cmder

La página oficial de cmdr es: https://cmder.app/.

Cmder sirve para mejorar drásticamente la experiencia de usar la línea de comandos en Windows. Su función principal es envolver el símbolo del sistema estándar (cmd.exe), PowerShell o Bash, proporcionando una interfaz moderna y funcional sin necesidad de instalar nada en el sistema operativo.

Es ideal para desarrolladores y administradores de sistemas que necesitan un entorno de terminal consistente que puedan llevar consigo en una memoria USB o usar en máquinas donde no tienen permisos de administrador para instalar software.

Cmder incorpora Clink, que añade las funcionalidades de Bash (el intérprete de línea de comandos predeterminado en la mayoría de las distribuciones de GNU/Linux y en macOS) a Windows.

La opción más completa y recomendada para reemplazar Cmder en Mac es iTerm2. Es el estándar de la industria para desarrolladores en macOS y ofrece funcionalidades que Cmder intenta emular en Windows.

Cmder no requiere instalación, es portable.

  • Ve a la página oficial de cmder y descarga la versión Full (que viene con Git integrado).
  • Descomprime el archivo .zip en la carpeta que desees y ejecuta Cmder.exe.

Agregar Cmder al PATH de Windows

Si quieres poder abrir Cmder desde cualquier lugar de tu computadora o integrarlo bien con tu sistema, sigue estos pasos:

  • Presiona Win + R, escribe sysdm.cpl y pulsa Enter.
  • Ve a la pestaña Opciones avanzadas.
  • Haz clic en el botón "Variables de entorno..." (abajo a la derecha).
  • En la sección "Variables de usuario", busca la que se llama Path y hazle doble clic.
  • Haz clic en "Nuevo" y pega la ruta de la carpeta donde has guardado Cmder (por ejemplo: C:\Cmder).
  • Dale a Aceptar a todo.

Ahora podrás ejecutar cmder desde cualquier carpeta de Windows.

Paso 3. Instalamos fnm. Fast Node Manager

fnm. Fast Node Manager, es un gestor de versiones de Node.js moderno, rápido y ligero. Permite instalar y cambiar entre diferentes versiones de Node.js según los archivos .nvmrc o .node-version del proyecto

  • Abre cmder, escribe la siguiente instrucción y presiona Enter. Si se te pide aceptar unos términos de Microsoft, escribe Y (de Yes) y presiona nuevamente Enter.

    winget install Schniz.fnm

  • Como Cmder es una consola especial, tenemos que decirle dónde buscar a fnm. Cierra Cmder y vuélvelo a abrir.

  • Escribe este comando en Cmder para abrir su archivo de configuración en el bloc de notas:

    notepad %CMDER_ROOT%\config\user_profile.cmd

  • Ve hasta el final de ese archivo de texto que se abrió, pega esta línea en una nueva línea después del @echo off:

    @FOR /F "tokens=*" %%z IN ('fnm env --use-on-cd') DO @%%z
  • Guarda el archivo (Ctrl + G o Archivo > Guardar), cierra el bloc de notas y cierra Cmder.

Paso 4. Instalar la última versión estable de Node.js

Cierra Cmder (si no lo has cerrado) por última vez y vuélvelo a abrir. para integrar completamente fnm. Ya podemos instalar la versión más estable de Node.js escribiendo la siguiente instrucción en Cmder:

fnm use --install-if-missing lts-latest

Paso 5. Instalar pnpm

Ejecuta este comando en tu Cmder:

npm install -g pnpm

Cuando termine de instalar escribe pnpm -v para confirmar que la instalación ha sido correcta. Si te devuelve su número de versión, ¡ya tenemos vía libre para comenzar a programar en Node.js

Tu entorno de desarrollo está impecable: Cmder como terminal, fnm gestionando Node.js y pnpm como el motor turbo para tus paquetes. Estás usando exactamente el mismo stack de herramientas que un desarrollador profesional avanzado.

Configurar cmder como terminal en VS Code

VS Code necesita que configuremos cmder explícitamente en su propia lista de perfiles para que aparezca en el menú desplegable de la terminal y poderlo seleccionar. Para ello:

  1. Presionamos Ctrl , en VS Code para abrir la Configuración.
  2. Buscamos el icono pequeño que parece un archivo con una flecha girando (Open Settings JSON). Clicamos sobre el mismo. Se abrirá un archivo de texto lleno de configuraciones.

  3. Al final de todo, antes de la última llave }, pegamos el siguiente bloque de código (cambiando C:\\Cmder por la ruta exacta donde está guardado tu Cmder: (las dobles barras diagonales \\, son obligatorias ya que representan un caracter de escape)

    
"terminal.integrated.profiles.windows": {
"Cmder": {
  "path": "cmd.exe",
  "args": ["/k", "C:\\Cmder\\vendor\\init.bat"]
}
},
"terminal.integrated.defaultProfile.windows": "Cmder"
  4. Guardamos el archivo con Ctrl S y abrimos la terminal de VS Code Ctrl Ñ
  5. Por defecto, VS Code suele abrir la terminal clásica de Windows (PowerShell o CMD). Para usar Cmder configurado con fnm dentro de VS Code, hacemos clic en la flechita que está al lado del botón + en la esquina de la terminal abajo a la derecha, y selecciona Cmder.

Cambiar el tamaño de la letra de la terminal

Para cambiar únicamente el tamaño de la letra de la terminal, sigue estos pasos:

  • Volvemos a presionar Ctrl , en VS Code para abrir la Configuración.
  • En la barra de búsqueda de arriba, escribimos: Terminal Font Size.
  • Se mostrará una opción que dice Terminal > Integrated: Font Size
    que por defecto viene en 14.
  • Escribe el tamaño de texto que te sea cómodo (se actualiza al instante).
